# mkrest-pages

## 项目简介

`mkrest-pages` 是一个用于归档和发布多个独立 HTML 页面的仓库。它不绑定任何特定域名或网站，仅作为一组静态页面的托管与维护空间。适合存放各类轻量级前端页面、演示页、工具页或其他无需后端支持的独立 HTML 文件。

## 目录说明

仓库根目录下存放所有 HTML 页面文件及其相关资源。目录结构建议如下：

```
mkrest-pages/
├── index.html          # 首页或索引页面（可选）
├── page1.html          # 示例页面
├── page2.html          # 示例页面
├── assets/             # 公共资源文件夹（可选）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

你可以根据实际需要调整目录结构，但请保持页面文件位于根目录，以方便直接通过仓库的 Pages 功能访问。

## 页面归档说明

本仓库中的每个 HTML 页面均为独立存档，互不依赖。页面可包含以下内容：

- 纯静态 HTML/CSS/JavaScript
- 外部 CDN 资源（需确保链接长期有效）
- 自包含的小型工具或演示

**不推荐**存放的内容：

- 需要后端接口支持的动态页面
- 包含用户隐私或敏感信息的页面
- 体积过大的媒体文件（建议使用外部图床）

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面时请保持目录结构清晰。
- 提交新页面时，建议在 commit 信息中简要说明页面用途。
- 如需删除页面，请确保该页面已不再被引用。
- 仓库默认开启 GitHub Pages 服务，根目录下的 `index.html` 将作为站点首页。
- 所有页面均为公开内容，请勿存放私人或机密资料。
